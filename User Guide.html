1. Abstract

Universal Resource Recommender (UReR) is a web platform able to expose and recommend to its users resources of interest, nearly available. These can be: known people or common places (e.g. shops, parks, railway stations etc.). The system uses different modules (micro-services) in order to implement the functionalities required and to give the possibility to expand its capabilities. The application also exposes a REST API that third parties can use to enhance their own works.

2. Introduction

UReR app exposes two main features: recommendations and alerts.

The recommendations part is the one that will take care of showing the user information about resources of interest nearly available. This part is also made up from two components: places and people. The places component will display on a map places that are near the user's current location. These places can be: shops, restaurants, museums, parks, monuments, cinemas etc. The initial places shown on the map will be related with the user's preferences, but the user will also have the possibility to filter the types of places shown. For example, if a user has as a preference the fact that he is interested in shopping, the app will display on the map only the shops that are near the location of the user. If the user wants to see more than shops on the map, he can select from a list of places types the ones that he wants to see (restaurants, parks, museums, etc.).

The people component will be responsible of displaying the list of friends or enemies the user currently has and also recommend a list of possible known people. These recommendations will be made based on the location of the users, their preferences and their common friends.

The alerts component will be responsible of sending alerts to the user on different situations. For example the user can receive an alert when a friend is nearby in order to go and say hi, or when an enemy is nearby in order to avoid him. Another alert example can be the following: the user is near a cinema and has as preferences the fact that he likes horror movies. Knowing this, the app can send an alert saying that the new horror movie "WADe Project Presentation" was released the other day and might be played at that cinema. The application can easily be extended to show other types of alerts.

In order to access the functionalities described above the user must be signed in. In order to login the user has two possibilities: use a QR code or a standard username and password.
